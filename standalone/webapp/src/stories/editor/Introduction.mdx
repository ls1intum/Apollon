# Editor stories

The editor (`@tumaet/apollon`) is a Tailwind-free library that wires its own
zustand stores and ReactFlow canvas internally. Its stories resolve from source
via the webapp's Vite aliases (`@tumaet/apollon` → `library/lib`) so they stay
in lockstep with the editor. The shared harness for every story below lives in
`src/stories/_support/editor.tsx`.

Each of the 13 diagram types gets its own story file (`Editor/Class Diagram`,
`Editor/BPMN`, …) with ~10–19 stories, plus the cross-cutting files
`Editor/Modes`, `Editor/Element Tags`, `Editor/Controls` and
`Editor/Playground`. The 13 canonical per-type fixtures —
which also drive the visual-regression suite — are exported from the harness as
`diagramFixtures` / `fixtureByType`.

## What the stories cover

Stories mount the editor (or a slice of it) through a small set of harness
components, each isolating a different surface. A full-editor mount needs a sized
container — the canvas renders blank at zero height — so those stories use
`layout: "fullscreen"`.

### Full editor

- **`<ApollonFixture>`** — a complete read-only editor from a serialized
  `UMLModel`. The canonical "show every diagram" path (and the GoF class-diagram
  templates). The editor wires its own stores/ReactFlow, so **no decorator is
  needed**.
- **`<ApollonEditable>`** — the fully editable editor: the element palette
  (Sidebar) shows, elements are selectable/movable, and clicking opens edit
  popups. Pass a `model` to edit a populated sample or a `type` for a blank
  canvas to build from scratch. This is the dominant per-type "Playground" /
  "Blank" surface.
- **`<ApollonAssessable>`** — the editor in **assessment** mode
  (`mode={ApollonMode.Assessment}`), the grading surface. Default is the
  _give-feedback_ workflow (click an element to score + comment); `readonly`
  switches to the _see-feedback_ review workflow over populated `assessments`.

### Highlights (quiz / interactive elements)

- **`<ApollonHighlightPicker>`** — opens the editor on the **Highlight** view,
  where an author marks elements interactive for an exam/quiz
  (`defaultView`/`availableViews` expose the Modelling↔Highlight toggle).
- **`<ApollonWithHighlights>`** — a read-only editor with **host-driven**
  highlights painted on via the imperative `editor.setElementHighlights(...)`
  overlay (id → CSS color), e.g. Athena suggestions or missing-feedback tints.
  The overlay is ephemeral — never serialized or shared.

### Galleries (elements & edges)

- **`<ElementGallery>` / `<ElementPreview>`** — render the SVG element renderers
  in isolation, the same call the Sidebar makes for its drag ghosts. These read
  the three zustand stores, so they're wrapped in `EditorStoreDecorator`
  (diagram / metadata / assessment-selection, built from one fresh `Y.Doc`).
  `elementConfigsFor(type)` returns every palette element config for a type.
- **`<EdgeGallery>` / `<EdgePreview>`** — every connector a diagram family
  draws, via the editor's own `EdgeTypePreviewIcon` (the exact mini-edge the
  edge-type dropdown shows). `edgeTypeCatalog` enumerates them.

### Edit & feedback popovers

- **`<SeededPopoverHarness>`** — renders a single edit or feedback popover's
  content in isolation. `seed` populates the diagram store with the node/edge the
  popover targets; the harness supplies the store contexts plus a hidden,
  controlled ReactFlow (so edge/BPMN popovers resolve their element) inside
  `.apollon-editor > .apollon-popover` for theme + popup styling. Drives the edit
  popovers and the **give-/see-feedback** popovers (`ClassGiveFeedbackPopover`,
  `ClassSeeFeedbackPopover`, the edge variants, …).

### Assessment-review selection

- **`<SelectionHarness>`** — the grading surface's multi-select. Mounts in
  `mode=Assessment` + `readonly` (the combo under which renderers read
  selection/hover from the assessment-selection store) and selects the given
  ids, so stories can assert single, nested, and hover selection styling.

### Modes (`Editor/Modes`)

A focused set proving the editor's non-default surfaces against the Class
fixture: the **Highlight** picker, the host-driven **highlight overlay**, the
assessment-review **selection** model (single / nested / hover), the
**Exporting** mode (static export — selection, popovers, and editing
affordances dropped), and the **dark** theme. These carry `play` assertions
where the rendered state is testable.

### Element tags (`Editor/Element Tags`)

Host-defined grouping labels on element data. Both halves: **authoring** tags on
a class and its members in the edit popover, and **coloring** a whole group —
looked up with `editor.getElementIdsByTag(tag)` — by a build result through the
ephemeral highlight overlay.

### Playground (`Editor/Playground`)

A single interactive editor with a live `readonly` toggle in assessment mode —
flip between the give-feedback workflow and the see-feedback review surface.
