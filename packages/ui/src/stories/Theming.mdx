# Theming

`@tumaet/ui` is themed entirely through CSS custom properties defined in
`src/styles/tokens.css`. There is no runtime `<style>` injection — the tokens
are static CSS so the first paint is always correct, and dark mode is a single
attribute flip.

## The three-layer token system

Tokens cascade through three layers. Each layer references only the layer above
it, which keeps the public theming surface small and lets external embedders
work standalone.

1. **Primitives — `--primitive-*`.** The brand/neutral palette. This is the
   ONLY layer where literal `#hex` / `rgb()` values live. Everything else
   references these.

2. **Editor theming API — `--apollon-*`.** The public, documented contract that
   embedders of `@tumaet/apollon` override. It references **primitives only**,
   never `--home-*`, so an external host that themes the editor never depends on
   app-level semantics.

3. **App semantics — `--home-*`.** The webapp's semantic tokens (surfaces,
   text, borders, accent, badges, tags, banners, toasts, radius). These
   reference `--apollon-*` and primitives, so re-theming the `--apollon-*` layer
   flows through to the app automatically.

```
--primitive-*  →  --apollon-*  →  --home-*
  (literals)      (public API)    (app semantics)
```

## The `data-theme` contract

Dark mode is a single mechanism: the `data-theme="dark"` attribute on the
document root (`:root` / `<html>`). Light values live in the `:root` block of
`tokens.css`; dark mode is expressed as a small set of **deltas** in
`:root[data-theme="dark"]` that re-point primitives and a few `--apollon-*` /
`--home-*` tokens. Because components consume the semantic tokens, the flip
re-themes everything without per-component dark variants.

Use the **theme toolbar toggle** above (top toolbar) to switch light / dark —
it is wired through `@storybook/addon-themes`' `withThemeByDataAttribute` with
`attributeName: "data-theme"`, matching the production contract exactly.

## Theming the editor from a host: `createApollonTheme`

Embedders override the public `--apollon-*` tokens. `createApollonTheme`
(from `src/theme.ts`) turns a typed, partial `ApollonTheme` object into a
framework-agnostic style object of `--apollon-*` custom properties you can
spread onto any element's `style`. Only the keys you provide are emitted, so
un-themed tokens keep their built-in light/dark fallbacks.

```tsx
import { createApollonTheme } from "@tumaet/ui"

const style = createApollonTheme({
  primary: "#ff5722",
  surface: "var(--my-brand-surface)",
  radius: "10px",
})

// style === { "--apollon-primary": "#ff5722", "--apollon-surface": "…", … }
;<Apollon theme={style} />
```

See `library/THEMING.md` for the full variable contract and host patterns, and
the **Tokens** page for a live swatch view of every layer (re-themes as you
toggle the toolbar).
